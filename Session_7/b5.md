# Kiến trúc 3 tầng – Hệ thống Blog Cá Nhân

## 1. Tổng quan hệ thống
Hệ thống blog cá nhân được chia thành 3 tầng chính theo kiến trúc 3 lớp (Three-Tier Architecture):
- **Presentation Layer**: Tầng giao diện người dùng.
- **Business Logic Layer**: Tầng xử lý nghiệp vụ.
- **Data Access Layer**: Tầng truy cập dữ liệu.

Mỗi tầng gồm nhiều **module**, tương ứng với các chức năng nghiệp vụ của hệ thống (bài viết, bình luận, người dùng).

---

## 2. Phân rã hệ thống thành các module

| **Tầng** | **Các module chính** | **Chức năng / Vai trò** |
|-----------|----------------------|--------------------------|
| **Presentation Layer** | `PostController`<br>`CommentController`<br>`UserController` | Giao diện người dùng (web hoặc mobile). Nhận yêu cầu từ người dùng và hiển thị kết quả. |
| **Business Logic Layer** | `PostService`<br>`CommentService`<br>`UserService` | Xử lý nghiệp vụ chính: tạo/chỉnh sửa bài viết, xử lý bình luận, xác thực người dùng. |
| **Data Access Layer** | `PostRepository`<br>`CommentRepository`<br>`UserRepository` | Quản lý truy cập dữ liệu (CRUD) trong cơ sở dữ liệu MySQL hoặc PostgreSQL. |
